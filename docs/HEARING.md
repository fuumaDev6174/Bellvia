# ヒアリング・議論記録（勤怠管理の改善）

日付: 2026-04-09

---

## 1. 休憩打刻の追加

### 要件
- 1日に複数回の休憩を記録できること
- 「本日 N 回目の休憩」を打刻画面に表示
- 3回目以降の休憩では警告バッジを表示（「休憩が多くなっています」）
- 管理者側の勤怠一覧にも休憩回数・合計休憩時間を表示

### 設計方針
- `attendance_breaks` テーブルを新設（`attendances` とは別テーブル）
- 1つの出勤記録に対して複数の休憩レコードを紐付け
- 勤務時間 = (clock_out - clock_in) - 合計休憩時間

### ボタン状態遷移
```
未出勤 → [出勤する] → 出勤中 → [休憩開始] → 休憩中 → [休憩終了] → 出勤中 → ... → [退勤する] → 退勤済
```

## 2. 打刻のデバイス・場所制限

### 課題
- 店舗外からの打刻を防ぎたい
- 店舗WiFiに接続している他人の端末からの打刻も防ぎたい

### 設計方針: 3層構造

| 層 | 対策 | 目的 | 解決すること |
|----|------|------|------------|
| ① | IP制限 | 場所の制限 | 店舗/会社ネットワーク外からの打刻を拒否 |
| ② | GPS | 場所の補強 | IP制限を補強。範囲外ならアラートを記録（即拒否ではなく記録） |
| ③ | セルフィー撮影 | 本人確認 | なりすまし打刻の防止。顔写真を保存 |

### IP制限の詳細
- `stores` テーブルに `allowed_ips text[]` を追加
- `companies` テーブルにも `allowed_ips text[]` を追加（会社全体の許可IP）
- 打刻APIでリクエスト元IPをチェック
- 店舗 or 会社の許可リストに含まれなければ拒否

### GPS の詳細
- 打刻時にブラウザの Geolocation API で位置情報を取得
- 店舗の座標（`stores` に `latitude` / `longitude` を追加）との距離を計算
- 一定距離（例: 500m）以上離れている場合はアラートを記録
- 即拒否ではなく、管理者に通知・記録する方式（外出先での正当な打刻も考慮）

### セルフィー撮影の詳細
- 出勤・退勤ボタン押下時にカメラを起動
- 顔写真を撮影し `attendance_photos` テーブル（または Supabase Storage）に保存
- 管理者は勤怠一覧から写真を確認可能
- 日常的なチェックは不要、疑わしい時だけ確認

## 3. なりすまし防止の議論

### WiFi問題
IP制限 + GPS だけでは「同じネットワーク・同じ場所にいる別人」を防げない。
→ セルフィー撮影が最も効果的。顔写真が残るリスクを嫌がるため、強力な抑止力になる。

### 検討した他の方式

| 方式 | 抑止力 | 実装コスト | 問題点 |
|------|:------:|:----------:|--------|
| セルフィー撮影 | **高** | 中 | カメラ許可が必要 |
| 個人PINコード | 中 | 低 | PIN共有で突破可 |
| ランダム確認質問 | 中 | 中 | 回答共有で突破可 |
| 給与明細表示（社会的抑止） | 中〜高 | 低 | 気にしない人には無効 |

### 決定
- **セルフィー撮影**を採用（物理的な証拠が残り、共有では突破不可能）

---

## 4. 決定事項まとめ

- 休憩: `attendance_breaks` テーブルで複数休憩対応 + 回数表示 + 3回目警告
- 場所制限: IP制限（店舗/会社） + GPS アラート記録
- 本人確認: セルフィー撮影で顔写真を保存
- 3層構造（IP + GPS + セルフィー）で進める
