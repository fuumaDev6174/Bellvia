# フェーズ別 機能一覧

凡例: ● コア（必須） / ◐ 重要（あると大きな差） / ○ Nice to Have / ◆ ヒアリング後に判断

---

## Phase 1: 集客導線 + 予約基盤【最優先】

### LP / Webサイト
- ● 店舗別LP — 3店舗それぞれのページ（メニュー・スタイリスト・アクセス）
- ● メニュー一覧 — カテゴリ分け・料金・所要時間
- ● スタイリスト紹介 — 写真・得意スタイル・指名予約への導線
- ◐ スタイルギャラリー — 施術事例の写真ギャラリー（Insta連携も視野）
- ○ 口コミ表示 — Google口コミの自動取得・LP上表示
- ● SEO基本設計 — meta/OGP/構造化データ/地域名キーワード

### Web予約システム
- ● 予約フォーム — 店舗・日時・メニュー・スタイリスト選択
- ● 空き状況カレンダー — リアルタイムの空き枠表示
- ● 予約確認メール/LINE通知 — 自動送信
- ◐ ゲスト予約 — 会員登録なしでも予約可能（ハードル最小化）
- ◐ リマインド通知 — 前日LINE/メール自動送信
- ○ キャンセル待ち — 空きが出たら自動通知

### 管理画面（最小構成）
- ● 予約一覧・カレンダー — 店舗別/スタイリスト別の予約表示
- ● 予約の手動追加・変更・キャンセル — 電話予約等の対応
- ● メニュー管理 — CRUD + 公開/非公開の切り替え
- ● スタイリスト管理 — 登録・出勤日設定
- ● 顧客一覧 — 基本情報 + 予約履歴

### 認証 / マルチテナント基盤
- ● ログイン/認証 — Supabase Auth（Email + 将来LINE Login）
- ● 会社→店舗→スタッフ構造 — RLSベースの権限制御
- ● ロール管理 — company_admin / store_manager / stylist

---

## Phase 2: 業務効率化 + 顧客管理強化【Phase 1安定後】

### 顧客カルテ
- ● 施術履歴 — 日時・メニュー・担当者・使用薬剤・仕上がり写真
- ◐ カウンセリングメモ — アレルギー・髪質・好み・NGなど
- ◐ 写真記録 — Before/After画像の保存・比較
- ○ 来店サイクル分析 — 次回来店予測・自動フォロー

### シフト管理
- ● シフト作成・公開 — カレンダーUI、スタッフごとの出勤設定
- ◐ 希望シフト提出 — スタッフが希望を出し、店長が確定
- ● 予約枠との自動連動 — シフトに基づいて予約可能枠を自動生成

### 売上管理
- ● 日次売上レポート — 施術完了→自動売上計上
- ◐ 決済種別管理 — PayPay/現金/カード等の記録
- ◐ スタイリスト別売上 — 個人の売上・指名数の集計
- ○ 会社ダッシュボード — 全店舗横断の売上・KPI一覧

### 在庫管理（ヒアリング後に詳細化）
- ◆ 商材在庫管理 — 薬剤・シャンプー等の在庫数管理
- ◆ 発注アラート — 在庫が閾値を下回ったら通知
- ◆ 消費量追跡 — 施術→商材消費の紐付け

---

## Phase 3: SNS・口コミ統合 + マーケティング【Phase 2並行 or 後】

### 口コミ・レビュー管理
- ● Google口コミ一元管理 — 取得・返信・分析
- ◐ 口コミ依頼の自動送信 — 施術後にLINE/メールで依頼
- ○ AI返信ドラフト — 口コミへの返信案を自動生成

### SNS連携
- ◐ Instagram投稿管理 — 管理画面から画像/動画投稿
- ○ 投稿スケジュール — 予約投稿・カレンダー管理
- ○ スタイルギャラリー連動 — カルテ写真→LP + Instaへ自動連携

### LINE活用強化
- ◐ セグメント配信 — 来店頻度・最終来店日でターゲティング
- ○ クーポン管理 — 作成・配信・使用追跡
- ○ LINE予約 — リッチメニューから直接予約

---

## Phase 4: 分析・高度化・SaaS展開準備【将来】

### 分析 / BI
- ○ 経営ダッシュボード — 売上推移・客単価・リピート率・LTV
- ○ 集客チャネル分析 — LP/LINE/Insta/紹介の流入比較
- ○ スタッフ生産性分析 — 稼働率・売上/時間

### SaaS化準備
- ○ セルフオンボーディング — 他のサロンが自分で登録・初期設定できる
- ○ プラン・課金管理 — Stripe連携の月額課金
- ○ テンプレートLP — 店舗情報入力だけでLP自動生成
