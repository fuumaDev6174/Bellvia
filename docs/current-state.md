# 現状ツールマップ（As-Is）

## 利用中ツール一覧

### ホットペッパービューティー（サロンボード）
- **用途:** 予約管理のみ
- **方針:** 置き換え（ホトペ脱却前提）
- **機能:** 予約受付、予約カレンダー、顧客情報（限定的）
- **備考:** 掲載料が発生。脱却のタイミングはヒアリングで確認。

### まいどおおきに
- **用途:** 社員管理、在庫管理、予約処理管理
- **方針:** 要ヒアリング（置き換え範囲を判断）
- **機能:** 社員管理、在庫管理、予約処理管理（サロンボードから手動入力）
- **備考:** 会社/店舗の区別なしで全体管理。使いにくいとの声あり。詳細は未把握。

### 口コミ管理システム（サービス名未確認）
- **用途:** LINE公式・Instagram・Google口コミの一元管理
- **方針:** 統合検討（自社実装 vs 継続利用）
- **機能:** LINE口コミ対応、Insta投稿（画像/動画）、Google口コミ対応
- **備考:** 具体的なサービス名・月額コストはヒアリングで確認。

### PayPay ダッシュボード
- **用途:** QR決済管理
- **方針:** 連携（API統合 or 記録のみ）
- **機能:** PayPay売上確認
- **備考:** 現金は完全に手動管理で集計が煩雑。

### LP（デザイン中）
- **用途:** ウェブ集客
- **方針:** 自社開発（Phase 1 のメイン）
- **機能:** デザインカンプのみ存在、集客導線なし
- **備考:** ここから予約への導線構築が最優先タスク。

## 主な課題
- 5つ以上のツールを横断 → スタッフの負担が大きい
- サロンボード→まいどおおきにへの手動転記 → 二重入力が最大の非効率
- 現金売上の手動管理 → 集計漏れリスク
- 会社/店舗の区別がない管理 → 3店舗のデータが混在
- LP→予約の導線がない → せっかくのデザインが集客に繋がっていない
