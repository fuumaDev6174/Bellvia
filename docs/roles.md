# ロール・権限設計

## ロール一覧

### company_admin（会社管理者）
- **スコープ:** 全店舗横断
- **権限:** 全データ閲覧・編集 / 店舗作成 / スタッフ管理 / 売上分析
- **想定ユーザー:** オーナー・経営者

### store_manager（店長）
- **スコープ:** 担当店舗のみ
- **権限:** 店舗内データ閲覧・編集 / シフト管理 / 在庫管理 / 売上確認
- **想定ユーザー:** 店長

### stylist（スタイリスト）
- **スコープ:** 自分の情報のみ
- **権限:** 自分の予約・シフト確認 / 施術記録入力 / 自分の売上確認
- **想定ユーザー:** スタイリスト

### guest（ゲスト）
- **スコープ:** 公開情報のみ
- **権限:** LP閲覧 / 予約フォーム利用 / 会員登録
- **想定ユーザー:** お客様（未登録）

### customer（顧客会員）
- **スコープ:** 自分の情報のみ
- **権限:** 予約履歴確認 / 予約変更・キャンセル / ポイント確認
- **想定ユーザー:** お客様（会員登録済み）

## 権限マトリクス

| 操作 | company_admin | store_manager | stylist |
|------|:---:|:---:|:---:|
| 全店舗データ閲覧 | ✅ | ❌ | ❌ |
| 自店舗データ閲覧 | ✅ | ✅ | ❌ |
| 自分のデータ閲覧 | ✅ | ✅ | ✅ |
| 予約作成・編集 | ✅ | ✅ | ❌ |
| メニュー管理 | ✅ | ✅ | ❌ |
| スタッフ管理 | ✅ | ✅（自店舗） | ❌ |
| シフト管理 | ✅ | ✅ | 閲覧のみ |
| 売上閲覧（全店） | ✅ | ❌ | ❌ |
| 売上閲覧（自店舗） | ✅ | ✅ | ❌ |
| 売上閲覧（自分） | ✅ | ✅ | ✅ |
| 店舗作成・編集 | ✅ | ❌ | ❌ |
| 施術記録入力 | ✅ | ✅ | ✅ |

## 実装方針
- Supabase RLS で company_id → store_id → user_id のチェーンで権限制御
- staff テーブルの role カラムで判定
- 将来の他社導入時にもマルチテナント対応済みとなる設計
