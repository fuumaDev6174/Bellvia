## 1. 公開ページ（顧客向け）

### トップページ (`/`)
- 全アクティブ店舗をカードグリッドで表示
- 各カード: 店舗名・住所・紹介文の抜粋
- カードクリック → 店舗ページへ遷移

### 店舗ページ (`/:storeSlug`)
- ヒーローセクション + 店舗情報
- おすすめメニュー（最大4件）、スタイリスト紹介（最大3件）
- 住所・電話番号・営業時間・紹介文
- 「予約する」CTAボタン
- SEO用の構造化データ（JSON-LD）出力

### メニュー一覧 (`/:storeSlug/menu`)
- カテゴリ別にグループ表示（カット、カラー、パーマ等）
- 各メニュー: 名前・料金・所要時間
- 「予約する」CTAボタン

### スタイリスト一覧 (`/:storeSlug/stylists`)
- グリッド表示（写真・名前・得意技術バッジ）

### Web予約フォーム (`/:storeSlug/reserve`)
6ステップのウィザード形式:

| Step | 内容 | 操作 |
|------|------|------|
| 1 | メニュー選択 | カテゴリバッジ付きリストから選択 |
| 2 | スタイリスト選択 | 指名 or 「おまかせ」 |
| 3 | 日付選択 | カレンダーUI |
| 4 | 時間帯選択 | 30分刻みのスロットグリッド（9:00〜21:00） |
| 5 | お客様情報 | 名前（必須）・電話番号（必須）・メール・備考 |
| 6 | 確認 | 内容確認 → 予約確定 |

- 前のステップの変更で後続の選択はリセットされる
- 空き状況はリアルタイムで取得（シフト + 既存予約から算出）
- 予約成功 → 確認ページへリダイレクト

### 予約完了ページ (`/:storeSlug/reserve/confirm`)
- チェックアイコン + 「予約が完了しました」
- 予約サマリー（店舗・メニュー・スタイリスト・日時・氏名）
- 「変更・キャンセルはお電話にて」の案内

## 2. 認証・ログイン

### ログインページ (`/admin/login`)
- メール + パスワード入力
- ログイン成功 → httpOnlyクッキーにトークン保存 → `/admin` へリダイレクト
- 失敗 → 「メールアドレスまたはパスワードが正しくありません」トースト
- staffレコードが無効（is_active=false）の場合もログイン不可

### セッション管理
- アクセストークン有効 → そのまま利用
- 期限切れ → リフレッシュトークンで自動更新
- 両方無効 → ログインページへリダイレクト

## 3. 管理画面レイアウト・ナビゲーション

### サイドバー構成

**company_admin（会社管理者）:**
```
ダッシュボード
└── 店舗管理（折りたたみ可）
    ├── 予約管理
    ├── メニュー管理
    ├── スタッフ管理
    ├── 売上管理
    ├── 在庫管理
    ├── 顧客管理
    ├── 勤怠管理
    └── 仕事量
────────────────
顧客一覧（全社）
売上総括
在庫総括
業種管理
```

**store_manager（店長）:**
```
ダッシュボード / 予約管理 / メニュー管理 / スタッフ管理
売上管理 / 在庫管理 / 顧客一覧 / 勤怠管理 / 仕事量
```

**stylist（スタイリスト）:**
```
ダッシュボード のみ
```

### ヘッダー機能
- **店舗切替**: company_adminは全店舗を切り替え可能（ドロップダウン）
- **ロールプレビュー**: company_adminが「店長として表示」「スタイリストとして表示」ボタンで他ロールのUIを確認可能。黄色バナーで「プレビュー中」表示
- **ログアウト**: クッキー削除 → ログインページへ

## 4. ダッシュボード

**ルート:** `/admin`  
**権限:** 全ロール

### 出退勤ボタン（全ロール共通）
- ヘッダー右側に配置
- 未出勤 → 緑「出勤する」ボタン
- 出勤中 → 赤「退勤する」ボタン + 出勤時刻表示
- 「メモを追加」トグルでメモ入力可

### スタッツカード

| ロール | カード1 | カード2 | カード3 |
|--------|---------|---------|---------|
| admin/manager | 本日の予約（確定数） | 今週の予約 | 顧客数 |
| stylist | 本日の自分の予約 | 次の予約（時間・顧客・メニュー） | — |

### タイムラインスケジュール
- 9:00〜21:00の時間軸（30分刻み）
- 行 = スタッフ、ブロック = 予約
- ブロック色: 青=確定、緑=完了、灰=キャンセル、赤=無断キャンセル
- 赤い縦線 = 現在時刻
- ホバーで詳細ツールチップ（時間・顧客名・メニュー名）
- stylistは自分の行のみ表示、admin/managerは全スタッフ表示

## 5. 予約管理

**権限:** company_admin, store_manager

### 予約一覧 (`/admin/reservations`)
- **フィルタ**: 開始日・終了日・ステータス・スタイリスト + リセットボタン
- **テーブル列**: 日時・顧客名・スタイリスト・メニュー・ステータスバッジ
- 行クリック → 詳細モーダル
- 「新規予約」ボタン → 作成モーダル

### 予約詳細モーダル
- 表示: ステータス・日時・顧客情報・メニュー（名前・料金・時間）・スタイリスト・備考
- **ステータス変更**: confirmed → completed / cancelled / no_show（一方向のみ）
- キャンセル・無断キャンセルは赤色のdangerスタイル

### 予約カレンダー (`/admin/reservations/calendar`)
- 週単位表示（前週・今週・翌週ナビゲーション）
- スタイリストごとにカード、7列（月〜日）
- 当日列はハイライト
- 各セルに予約の時間・顧客名・メニュー名・ステータスバッジ

## 6. メニュー管理

**ルート:** `/admin/menus`  
**権限:** company_admin, store_manager

### 一覧テーブル
| 列 | 内容 |
|----|------|
| メニュー名 | テキスト |
| カテゴリ | バッジ |
| 料金 | ¥表示（右寄せ） |
| 所要時間 | 「30分」等 |

| 公開 | トグルスイッチ（青=公開、灰=非公開） |

### メニュー作成・編集モーダル
| フィールド | 型 | 必須 | 備考 |
|-----------|-----|:----:|------|
| メニュー名 | text | ○ | |
| カテゴリ | select | | カット/カラー/パーマ/トリートメント/スパ/その他 |
| 説明 | textarea | | 3行 |
| 料金（円） | number | ○ | min=0 |
| 所要時間（分） | number | ○ | min=1 |
| 仕事量ポイント | number | | step=0.1, min=0, デフォルト=1.0 |
| 公開する | checkbox | | デフォルト=true |

### 削除
- 確認ダイアログ:「「{名前}」を削除しますか？この操作は取り消せません。」

## 9. 店舗管理

**権限:** company_admin のみ

### 店舗一覧 (`/admin/stores`)
- **フィルタ**: キーワード検索・業種ドロップダウン・公開状態
- 業種別にグループ表示（カラーバッジ付き）
- 店舗カード: 名前・ステータスドット・住所・電話番号

### 店舗詳細 (`/admin/stores/:storeId`)
5つのタブ:

| タブ | 内容 |
|------|------|
| 概要 | 基本情報（インライン編集）、営業時間（曜日別）、業種選択、公開/非公開トグル |
| スタッフ | 所属スタッフ一覧 |
| 顧客 | 店舗別顧客（検索付き） |
| 売上 | 日付フィルタ + 売上サマリー + 一覧テーブル |
| 在庫 | カテゴリ別在庫 + インライン数量編集 |

## 10. 売上管理

### 店舗別売上 (`/admin/sales`)
**権限:** company_admin, store_manager

- **PayPayインポートボタン**: ヘッダー右側
- **日付フィルタ**: 月初〜本日がデフォルト
- **サマリーカード**: 売上合計・客単価・決済種別内訳
- **一覧テーブル列**: 日時・顧客・担当・決済方法・金額
- 決済ラベル: 現金 / PayPay / カード / その他

### 売上総括 (`/admin/sales-overview`)
**権限:** company_admin のみ
- 全店舗横断の売上 + 店舗別内訳

### PayPayインポートモーダル
1. **ファイル選択**: CSVファイルをアップロード（.csv）
2. **自動解析**: Shift-JIS / UTF-8 自動判定。ヘッダー名で列を自動検出（決済日時・決済番号・決済金額・決済ステータス）
3. **プレビュー**: 取込対象件数・スキップ件数・合計金額を表示。テーブルでスキップ行はグレーアウト
4. **インポート実行**: 重複（取引ID一致）は自動スキップ。完了ステータス以外もスキップ。結果サマリーをトースト表示

## 11. 在庫管理

### 店舗別在庫 (`/admin/inventory`)
**権限:** company_admin, store_manager

- 在庫少のアラートバナー（黄色、件数表示）
- カテゴリ別にグループ表示
- **テーブル列**: 商品名・単位・在庫数・最低在庫・原価・状態バッジ・操作

### 在庫総括 (`/admin/inventory-overview`)
**権限:** company_admin のみ
- 全店舗横断の在庫一覧（読み取り専用）

## 12. 勤怠管理

### ダッシュボードの打刻ボタン
**権限:** 全ロール

| 状態 | ボタン | 表示 |
|------|--------|------|
| 未出勤 | 緑「出勤する」 | — |
| 出勤中 | 赤「退勤する」 | 「HH:MM から出勤中」 |

- 「メモを追加」トグルでメモ入力（任意）

### 勤怠管理ページ (`/admin/attendance`)
**権限:** company_admin, store_manager

- **フィルタ**: 開始日・終了日・スタッフ選択
- **サマリーカード**: スタッフ別の合計勤務時間（HH:MM）+ 出勤日数

**テーブル列:**
| 列 | 内容 |
|----|------|
| 日付 | 出勤日 |
| スタッフ | 氏名 |
| 出勤 | HH:MM |
| 退勤 | HH:MM or 「-」 |
| 勤務時間 | HH:MM（clock_out - clock_in） |
| ステータス | 出勤中（青）/ 完了（緑）/ 修正済（黄） |
| メモ | 出勤メモ / 退勤メモ / 修正理由 |
| 操作 | 編集ボタン |

### 勤怠修正モーダル
- 対象スタッフ名を表示
- 出勤時刻・退勤時刻をdatetime-localで編集
- **修正理由（必須）**: 空欄の場合「修正理由は必須です」エラー
- 保存すると status=corrected、corrected_by に修正者IDを記録

## 13. 仕事量

**ルート:** `/admin/workload`  
**権限:** company_admin, store_manager

- **フィルタ**: 開始日・終了日・スタッフ選択
- **サマリーカード**: 合計ポイント・施術件数
- **テーブル列**: スタッフ名・件数・合計ポイント・メニュー内訳

**メニュー内訳の表示:**
```
カット ×5 (5.0pt)  カラー ×3 (6.0pt)  トリートメント ×2 (1.0pt)
```
- 灰色バッジで各メニューの件数とポイントを表示

**計算ロジック:**
- 対象: ステータスが「completed」の予約のみ
- ポイント: 各メニューの `workload_points` フィールド（デフォルト1.0）
- 合計 = 完了予約のメニューポイントの総和

## 14. 業種管理

**ルート:** `/admin/business-types`  
**権限:** company_admin のみ

- 店舗に割り当てるカテゴリの管理
- **追加フォーム**: 業種名入力 + カラーピッカー（10色から選択）
- **一覧**: カラードット + 業種名 + 無効バッジ + 編集/削除ボタン
- **編集**: インラインで名前とカラーを変更
- **削除**: 確認ダイアログ「紐付いている店舗の業種は未設定になります」
